## Goal
Display the C major scale (C, D, E, F, G, A, B) as glowing spheres on the bass guitar fretboard. Notes appear sequentially (one by one in ascending order across all strings), and once the full sequence completes, all fade out and the cycle restarts in a continuous loop.

### Step 1: Define Note Data Model

Create a data structure mapping each string × fret combination to its note name. Only include notes in the C major scale (C, D, E, F, G, A, B — i.e., no sharps/flats).

```scala
// Chromatic scale starting from C
val chromaticNotes = Array("C", "C#", "D", "D#", "E", "F", "F#", "G", "G#", "A", "A#", "B")
// Open string MIDI-like semitone offsets from C: E=4, A=9, D=2(+12), G=7(+12)
// Actually simpler: open string note indices in chromatic scale
val openStringNotes = List(4, 9, 2, 7) // E, A, D, G (semitones from C)
val cMajorNotes = Set("C", "D", "E", "F", "G", "A", "B")
```

For each string (0–3) and fret (0–20), compute:
```
noteIndex = (openStringNote + fret) % 12
noteName = chromaticNotes(noteIndex)
if cMajorNotes.contains(noteName) → include in display list
```

This gives us a `List[(stringIndex, fretNum, noteName)]` — the full set of scale positions.

### Step 2: Create Glowing Sphere Meshes for Each Note Position

For each note position:
- Create a `SphereGeometry(0.08, 16, 16, ...)` (small sphere)
- Use `MeshStandardMaterial` with:
  - `color` based on note name (C=red, D=orange, E=yellow, F=green, G=cyan, A=blue, B=purple) for visual distinction
  - `emissive` = same color (for glow effect)
  - `emissiveIntensity` = 0.8
- Position: X from `stringXPositions(stringIndex)`, Y = `neckHeight/2 + 0.2` (above fretboard), Z = midpoint between fret(n-1) and fret(n) for fret > 0, or just behind nut for fret 0.
- Initial `scale.set(0, 0, 0)` — invisible at start.

Store all note meshes in an ordered `js.Array[Mesh]` (sorted by string low→high, then fret low→high for ascending sequence).

### Step 3: Animation State Machine

State variables:
```scala
var animPhase: Int = 0        // 0 = revealing, 1 = holding, 2 = fading out
var currentNoteIndex: Int = 0 // which note is being revealed
var phaseTimer: Double = 0.0  // accumulated time in current phase/note
val revealInterval = 0.15     // seconds between each note reveal
val holdDuration = 2.0        // seconds all notes stay visible
val fadeOutDuration = 0.5     // seconds for all notes to shrink/fade out
```

### Step 4: Animation Loop Logic

In the existing animation callback (line 169):

```
Phase 0 (Revealing):
  - phaseTimer += delta
  - While phaseTimer >= revealInterval:
    - noteMeshes(currentNoteIndex).scale.set(1, 1, 1) (snap to full size, or animate with lerp)
    - currentNoteIndex += 1
    - phaseTimer -= revealInterval
  - If currentNoteIndex >= totalNotes: switch to phase 1, reset timer

Phase 1 (Holding):
  - phaseTimer += delta
  - Optional: gentle pulse animation (scale oscillates between 0.9 and 1.1 using sin)
  - If phaseTimer >= holdDuration: switch to phase 2, reset timer

Phase 2 (Fading out):
  - phaseTimer += delta
  - progress = phaseTimer / fadeOutDuration (0→1)
  - All note meshes: scale.set(1 - progress, 1 - progress, 1 - progress)
  - If progress >= 1.0: reset all scales to 0, reset to phase 0, currentNoteIndex = 0
```

For smoother reveal, instead of snapping scale to 1, we can lerp each note's scale from 0 toward 1 over a short duration (e.g., 0.2s) using per-note timers. However, keeping it simple with a snap + slight overshoot (scale to 1.2 then back to 1.0 via lerp in subsequent frames) might look good.

**Refined approach**: Use per-note `revealedAt` timestamp. Each frame, for each note, compute how long since it was revealed and lerp scale from 0→1.2→1.0 using an ease-out curve. This gives a smooth "pop-in" effect.

## Animation Detail: Smooth Pop-In

For each note mesh, track when it was "triggered" (a `Double` timestamp from the clock's elapsed time). Each frame:
```
timeSinceReveal = elapsedTime - triggeredAt
if timeSinceReveal < 0 → scale = 0 (not yet revealed)
if timeSinceReveal < popDuration (0.2s):
  t = timeSinceReveal / popDuration
  // Overshoot ease: scale = 1.0 + 0.3 * sin(t * PI) — peaks at 1.3 midway, settles at 1.0
  s = 1.0 + 0.3 * sin(t * PI) * (1 - t) // actually just use: cubic ease out with overshoot
  // Simpler: s = min(1.2, t * 2) for first half, then lerp back to 1.0
  scale.set(s, s, s)
else → scale.set(1, 1, 1)
```

## Technical Considerations

1. **Opacity vs Scale for animation**: The facade doesn't expose `opacity`/`transparent` as typed properties. We have two options:
   - Use `mesh.asInstanceOf[js.Dynamic].material.opacity = x` and `.material.transparent = true` (dynamic access)
   - Use `scale` animation exclusively (grow from 0 → 1 for appear, shrink 1 → 0 for disappear)
   
   **Decision**: Use **scale animation** for simplicity and guaranteed compatibility. No opacity needed.

2. **Emissive glow**: `MeshStandardMaterial` supports `emissive` and `emissiveIntensity` at construction. For the glowing effect, set emissive to the note color. We may want to dynamically adjust `emissiveIntensity` — this will require `js.Dynamic` cast. Alternative: just keep static emissive and rely on the pop-in scale animation for visual impact.

   **Decision**: Set static `emissive` + `emissiveIntensity` at construction. If dynamic adjustment is needed later, use `js.Dynamic`.

3. **Note ordering for sequence**: Sort ascending by string (E=0 → G=3) then by fret (0 → 20). This gives a natural low-to-high progression.

4. **Fret 0 (open string) position**: Place the sphere slightly behind the nut, at Z = `-scaleLength/2 - 0.2` or between nut and the "virtual fret 0" position. Actually for fret 0, place it at `Z = -scaleLength/2 + someSmallOffset` (just past the nut on the playing side), roughly where you'd see an open-string indicator.

   Better: For fret 0, position at `Z = -scaleLength/2 - 0.3` (behind the nut, as a visual indicator of "open string").

5. **Color palette per note** (for visual distinction):
   - C = `0xff3333` (red)
   - D = `0xff8800` (orange)
   - E = `0xffff00` (yellow)
   - F = `0x33cc33` (green)
   - G = `0x00cccc` (cyan)
   - A = `0x3366ff` (blue)
   - B = `0x9933ff` (purple)
