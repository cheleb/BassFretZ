# Plan: Show Note + Intervals on the Bass Neck

## Goal

Transform the current "animated full-scale reveal" into an interactive tool where the bass player:

1. Picks a **root note** from a dropdown.
2. Toggles which **intervals** (root, 3rd, 5th, 7th, etc.) should be highlighted.
3. Sees all matching positions on the bass neck at once (static display), with the root note visually emphasised.

## Current State (Summary)

- `src/main/scala/dev/cheleb/mythreeapp/Main.scala` hardcodes `rootKey = "C"` and `scaleType = "major"`, computes all scale positions via `Scale.computeScalePositions`, pushes spheres into the scene, and drives an `Animation` that reveals/holds/fades the notes in a loop.
- `src/main/scala/dev/cheleb/mythreeapp/Scale.scala` has `chromaticNotes`, `noteIndexMap`, `scaleIntervals`, `openStringNotes` (E/A/D/G), `stringXPositions`, `noteColors`, `getScaleNotes`, `getFretZPosition`, `computeScalePositions`.
- `src/main/scala/dev/cheleb/mythreeapp/Animation.scala` drives the reveal/hold/fade cycle; will become unused for this feature.
- `src/main/scala/dev/cheleb/mythreeapp/Neck.scala` builds the 3D neck (fretboard, nut, frets, strings, markers).
- `index.html` has a single `<div id="app">` where the renderer is attached.

## Design Decisions (confirmed with user)

- **Input:** HTML UI — dropdown for root note + checkboxes for intervals. Re-renders live.
- **Intervals exposed:** common chord/arpeggio intervals:
  - Root (0), minor 2nd (1), major 2nd (2), minor 3rd (3), major 3rd (4), perfect 4th (5), tritone (6), perfect 5th (7), minor 6th (8), major 6th (9), minor 7th (10), major 7th (11), octave (12).
  - Defaults selected: Root, major 3rd, perfect 5th, minor 7th (a dominant 7 arpeggio — a very common bass shape). Adjustable.
- **Visual distinction:** root note = larger sphere (e.g. radius 0.14) with bright emissive outline; intervals = smaller spheres (radius 0.08) colored by interval type (not by note name).
- **Animation:** static display. The existing reveal/hold/fade `Animation` class is no longer used by `Main`; we keep the file on disk for now but stop wiring it in.

## Changes

### 1. `Scale.scala` — new helpers for intervals

Add (leave existing API intact so `Animation.scala` still compiles):

- `val commonIntervals: List[(Int, String, String)]` — ordered list of `(semitones, shortLabel, longLabel)`:
  - `(0, "R",  "Root")`
  - `(1, "b2", "Minor 2nd")`
  - `(2, "2",  "Major 2nd")`
  - `(3, "b3", "Minor 3rd")`
  - `(4, "3",  "Major 3rd")`
  - `(5, "4",  "Perfect 4th")`
  - `(6, "b5", "Tritone")`
  - `(7, "5",  "Perfect 5th")`
  - `(8, "b6", "Minor 6th")`
  - `(9, "6",  "Major 6th")`
  - `(10, "b7", "Minor 7th")`
  - `(11, "7",  "Major 7th")`
  - `(12, "8",  "Octave")`
- `val intervalColors: Map[Int, Int]` — distinct color per semitone (reuse palette from `noteColors` ideas; each interval gets one stable color regardless of root).
- `def computeIntervalPositions(rootKey: String, selectedSemitones: Set[Int], maxFret: Int = 20): List[(Int, Int, Int, String)]`
  - Returns `(stringIdx, fret, semitoneOffset, noteName)` tuples for every neck position whose note name equals `chromaticNotes((noteIndexMap(rootKey) + semitone) % 12)` where `semitone` is in `selectedSemitones`.
  - Octave (12) is treated as offset `0` for mapping but kept as a distinct visual tag so the UI colors octaves separately from the root.
  - Sorted by `(stringIdx, fret)`.

### 2. `Main.scala` — rewire to interactive, static rendering

- Remove the `Animation` construction and the reveal/hold/fade wiring.
- Keep scene/camera/renderer/controls/lights/`Neck.create()` setup.
- Replace the HUD `#note-display` content with a **control panel** (see section 3) and a small legend.
- Introduce mutable selection state:
  - `var rootKey: String = "C"`
  - `var selectedIntervals: Set[Int] = Set(0, 4, 7, 10)` (dominant-7 arpeggio default).
- Extract note-mesh creation into a local helper `renderSelection(): Unit`:
  1. Remove all previously added sphere meshes from `neckGroup` and dispose of their geometry/material.
  2. Compute positions via `Scale.computeIntervalPositions(rootKey, selectedIntervals)`.
  3. For each position, create a sphere:
     - If `semitoneOffset == 0` → "root" sphere: radius `0.14`, bright emissive color (use `Scale.noteColors(rootKey)` or a dedicated root color), higher `emissiveIntensity`, and add a thin ring/outline child (e.g. a slightly larger transparent `MeshBasicMaterial` sphere with `side = BackSide` to fake an outline, or a `RingGeometry` flat marker beneath it — simplest: a secondary sphere at 1.25x scale with `MeshBasicMaterial` wireframe).
     - Else → "interval" sphere: radius `0.08`, colored via `Scale.intervalColors(semitoneOffset)`.
  4. Set scale to 1 immediately (no animation); keep the spheres in `neckGroup`.
  5. Update the HUD legend to reflect which intervals are selected and their colors.
- Replace the animation loop body so it only calls `controls.update()` and `renderer.render(...)` (no per-frame scale mutation).
- Call `renderSelection()` once at startup, and again whenever the UI changes the selection.

### 3. HTML UI panel (built from Scala via `dom` APIs)

Create in `Main.scala` (no change to `index.html`):

- A fixed-position panel (top-left, replacing current `note-display`) containing:
  - **Root note** `<select>` listing all 12 notes from `Scale.chromaticNotes` (use flat/sharp forms consistent with existing map). Default `"C"`.
  - **Intervals** section: one `<label><input type=checkbox></label>` per entry in `Scale.commonIntervals`, each showing the short label and the color swatch of that interval. Defaults checked: 0, 4, 7, 10.
  - A small **legend** area that lists every currently-selected interval with its color and the resolved note name (e.g. "R — A", "3 — C#", "5 — E", "b7 — G").
- Event handlers:
  - `select.addEventListener("change", …)` → update `rootKey`, re-render selection + legend.
  - Each checkbox `change` → update `selectedIntervals`, re-render selection + legend.
- Keep the bottom "Drag to rotate • Scroll to zoom" hint.

### 4. `Animation.scala`

Not edited in this plan. It remains on disk but is no longer referenced from `Main`. If the compiler complains about unused warnings we leave it alone (Scala.js doesn't error on unused classes). No public API change.

## Validation

- `sbt` compile of the Scala.js module (module `my-three-app` — confirm via Metals `list-modules` during implementation) to ensure the new `Scale` members resolve and `Main` compiles.
- Manual smoke check (described, not executed):
  - Page loads, default selection shows A dominant-7 arpeggio shape across the neck when the user picks root = A and defaults remain.
  - Changing the dropdown or any checkbox re-renders the spheres without leftover meshes.
  - Root sphere is visibly bigger/brighter than interval spheres.
  - Legend updates to match current selection.

## Non-goals / Out of scope

- Playing audio for notes.
- Saving selection to localStorage / URL.
- Rewriting or deleting `Animation.scala`.
- Changing the 3D neck geometry, camera, or lighting.
- Fancy label rendering in the 3D scene (intervals communicated via color + legend only).
